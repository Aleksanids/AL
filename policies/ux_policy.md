# UX Policy

Use this only when the task touches UI, CLI UX, user-facing copy, forms,
tables, dashboards or error messages.

## Check

- Primary user workflow is visible and clear.
- System status is observable: loading, success, warning, error.
- Empty states are useful.
- Error recovery is possible.
- Keyboard navigation is not broken.
- Text is readable and does not overflow.
- Forms and tables support repeated work.
- Visual hierarchy helps scanning.
- Responsive behavior is considered for UI tasks.

## Reporting

If UI was in scope, report what was visually/behaviorally checked. If not
checked, say why.
