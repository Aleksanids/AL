# Documentation Handoff

## Procedure

1. Обновляй docs только в зоне, которую реально изменил.
2. Фиксируй факты: files, commands, outcomes, constraints.
3. Разделяй done, not run, risks, next step.
4. Не переписывай историю проекта и не придумывай неподтвержденные решения.
5. Для повторяемых процессов добавляй короткий checklist или command block.

## Output

- concise docs patch;
- clear commands;
- known limitations;
- next safe action.
