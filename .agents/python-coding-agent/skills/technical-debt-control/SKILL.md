# Technical Debt Control

## Rules

- Do not fix unrelated debt inside a narrow task.
- Record debt as finding, risk or next step.
- Fix debt only when it blocks the requested work or clearly reduces current
  patch risk.
- Prefer small tracked follow-up items.

## Output

- debt item;
- impact;
- suggested owner/path;
- safe next action.
