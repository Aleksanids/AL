# Feature Development

## Workflow

1. Define user-visible behavior and acceptance criteria.
2. Find existing extension points.
3. Plan data model, errors and side effects.
4. Implement minimal vertical slice.
5. Add unit tests and relevant docs.
6. Run quality gates.

## Acceptance

- Feature works for stated cases.
- Existing behavior unchanged.
- Tests cover normal and edge paths.
- Docs or examples updated when user-facing behavior changed.
