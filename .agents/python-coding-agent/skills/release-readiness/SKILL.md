# Release Readiness

## Checklist

- Acceptance criteria closed.
- Required quality gates pass.
- Docs updated.
- Version/changelog considered if package behavior changed.
- Security and secrets checks pass.
- Known warnings listed.
- Rollback or recovery path named.

## Output

`PASS`, `WARN` or `FAIL` with evidence.
