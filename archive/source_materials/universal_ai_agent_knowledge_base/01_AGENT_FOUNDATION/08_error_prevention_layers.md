# Слои предотвращения ошибок

## 13 слоёв защиты

1. Структурированная задача.
2. Acceptance criteria.
3. Must-read context.
4. Allowed/forbidden paths.
5. План до кода.
6. Small patch policy.
7. Static analysis.
8. Type checking.
9. Tests.
10. Security scan.
11. Diff review.
12. Critic review.
13. Verifier report.

## Почему это работает

LLM может ошибаться. Но если ошибка проходит через 13 независимых фильтров, вероятность попасть в main резко снижается.
