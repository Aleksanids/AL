# Жизненный цикл задачи

## 1. Intake

Агент принимает задачу и выделяет:

- цель;
- тип задачи;
- ограничения;
- входные данные;
- критерии приёмки;
- риски;
- ожидаемый артефакт.

## 2. Planning

План должен содержать:

- гипотезу причины;
- список файлов для чтения;
- список файлов для изменения;
- список файлов, которые нельзя менять;
- проверки;
- rollback strategy.

## 3. Implementation

Изменения должны быть минимальными и проверяемыми.

## 4. Verification

Проверка должна опираться на команды, тесты, diff и воспроизводимые факты.

## 5. Reporting

Отчёт должен отвечать:

- что изменено;
- зачем;
- где;
- чем проверено;
- какие риски остались.
