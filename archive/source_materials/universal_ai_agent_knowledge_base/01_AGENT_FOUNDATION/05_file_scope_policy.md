# File scope policy

## Цель

Запретить агенту менять лишние файлы.

## Поля задачи

```yaml
allowed_paths:
  - src/
  - tests/
forbidden_paths:
  - .env
  - secrets/
  - dist/
  - user_data/
```

## Правила

1. Если файл вне `allowed_paths`, агент не имеет права его менять.
2. Если файл попадает в `forbidden_paths`, изменение блокируется всегда.
3. Если задача требует расширить scope, агент останавливается и объясняет причину.
4. Hotfix не может превращаться в рефакторинг.
5. UI-задача не должна менять backend без отдельного основания.
