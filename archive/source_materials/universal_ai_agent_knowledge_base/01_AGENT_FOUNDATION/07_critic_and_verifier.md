# Critic и Verifier

## Critic

Critic ищет проблемы:

- задача решена частично;
- diff не соответствует цели;
- изменены лишние файлы;
- нет тестов;
- сломаны edge cases;
- исправление маскирует ошибку;
- есть security/UX regression.

Статус critic по умолчанию: `failed`.

## Verifier

Verifier принимает только по фактам:

- команды запущены;
- тесты пройдены;
- diff понятен;
- acceptance criteria закрыты;
- риски явно описаны.

## Запрет

Implementer не может сам принимать свой код.
