# Clean code для AI-агента

## Цель

Код должен быть понятным, проверяемым и изменяемым.

## Правила

1. Одна функция — одна понятная ответственность.
2. Названия отражают предметную суть, а не техническую догадку.
3. Сложная логика выносится в именованные функции.
4. Комментарии объясняют «почему», а не пересказывают «что».
5. Ошибки обрабатываются явно.
6. Магические числа заменяются константами.
7. Не использовать глобальное состояние без необходимости.
8. Не смешивать domain/application/infrastructure/UI.
9. Не прятать ошибки широким `except Exception`.
10. Не делать рефакторинг вместе с исправлением бага без отдельной причины.

## AI-specific

Если агент не понимает код — он должен сначала объяснить текущую логику, а не переписывать её.
