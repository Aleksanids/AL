# Стандарт code review

## Что проверять

1. Дизайн решения.
2. Соответствие задаче.
3. Простота.
4. Тесты.
5. Именование.
6. Ошибки и edge cases.
7. Безопасность.
8. Производительность.
9. Документация.
10. Совместимость.

## Правило reviewer

Ревью должно улучшать здоровье кодовой базы. Не надо блокировать из-за вкусовщины, но надо блокировать из-за риска, непроверенности и скрытой сложности.

## Для AI-агента

Critic должен быть строже implementer. Если нет доказательств — статус `failed` или `passed_with_warnings`.
