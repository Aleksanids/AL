# Small changes policy

## Принцип

Маленький diff лучше большого умного diff.

## Ограничения

- hotfix: до 3 файлов, если возможно;
- feature: разбивать на этапы;
- refactor: отдельная задача;
- tests: можно добавлять рядом с изменением;
- formatting whole repo запрещён без отдельной задачи.

## Признаки опасного изменения

- затронуто много несвязанных файлов;
- удалены тесты;
- изменены public API;
- добавлены зависимости;
- изменён CI;
- изменена архитектура без design note.
