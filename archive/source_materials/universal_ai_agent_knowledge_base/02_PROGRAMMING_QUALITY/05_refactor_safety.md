# Безопасный рефакторинг

## Разделять задачи

Нельзя смешивать:

- исправление бага;
- изменение поведения;
- переименование;
- архитектурный перенос;
- форматирование.

## Safe refactor checklist

1. До рефакторинга есть тесты.
2. Зафиксировано текущее поведение.
3. Изменение маленькими шагами.
4. После каждого шага тесты проходят.
5. Public API не изменён или изменение явно описано.
6. Производительность не ухудшена без причины.

## Запрет

Если агент не может объяснить, почему поведение не изменилось, рефакторинг не принимается.
