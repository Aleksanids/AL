# Ошибки и логирование

## Ошибки

- использовать конкретные исключения;
- не глотать исключения;
- не возвращать `None` вместо ошибки без явного контракта;
- сохранять причину ошибки;
- в UI показывать понятное сообщение;
- в логах оставлять технические детали.

## Логи

Лог должен отвечать:

- что произошло;
- где;
- с каким идентификатором операции;
- что делать дальше.

## Запрет

Нельзя логировать секреты, токены, пароли, cookies, персональные данные без необходимости.
