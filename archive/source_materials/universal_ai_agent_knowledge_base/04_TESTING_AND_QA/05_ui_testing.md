# UI testing

## Проверять

- главный пользовательский сценарий;
- пустые состояния;
- loading states;
- ошибки;
- keyboard navigation;
- responsive layout;
- focus order;
- кликабельность;
- недоступные действия;
- визуальные регрессии.

## Agent rule

Если задача меняет UI, без UX/test review задача не принимается.
